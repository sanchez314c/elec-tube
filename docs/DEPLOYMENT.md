# Deployment

## Distribution Targets

ElecTube targets Linux x86_64 desktops. It packages into two formats:

| Format | Command | Use Case |
|--------|---------|----------|
| AppImage | `npm run package:linux` | Portable, no install needed, runs on any Linux distro |
| deb | `npm run package:linux` | Debian/Ubuntu package, installs via `dpkg -i` or `apt` |

Both formats are generated by `electron-builder` and output to the `release/` directory.

## Build for Release

```bash
# Full build + package
npm run package

# Linux-only
npm run package:linux
```

This runs the full pipeline:
1. `vite build` compiles the renderer
2. `tsc -p tsconfig.main.json` compiles the main process
3. `electron-builder` packages everything

## Release Process

1. Update the version in `package.json`
2. Update `CHANGELOG.md` with the new version and changes
3. Update `VERSION_MAP.md` with the new version
4. Run `npm run package:linux`
5. Test the AppImage on a clean system
6. Create a GitHub release with the AppImage and deb as assets
7. Tag the release in git: `git tag -a v1.x.x -m "Release v1.x.x"`

## Environment Configuration

The packaged app reads these environment variables at runtime:

| Variable | Required | Purpose |
|----------|:---:|---------|
| `YOUTUBE_API_KEY` | Yes | YouTube Data API v3 access |
| `YOUTUBE_CLIENT_ID` | No | OAuth 2.0 for authenticated features |
| `YOUTUBE_CLIENT_SECRET` | No | OAuth 2.0 for authenticated features |
| `ELECTUBE_MPV_PATH` | No | Custom path to mpv binary |
| `ELECTUBE_YTDLP_PATH` | No | Custom path to yt-dlp binary |

Users must export `YOUTUBE_API_KEY` in their shell profile before launching the packaged app.

## Binary Dependencies

The development setup (`run-source-linux.sh`) auto-downloads mpv and yt-dlp to `bin/`. The packaged app does NOT include these binaries. Users of the packaged app need:

- **mpv** installed system-wide (`sudo apt install mpv`)
- **yt-dlp** installed system-wide (`pip install yt-dlp` or download the binary)

Or set `ELECTUBE_MPV_PATH` and `ELECTUBE_YTDLP_PATH` to point to the binaries.

## Post-Deployment Verification

After installing the packaged app:

1. Launch ElecTube
2. Verify the trending feed loads (confirms API key is working)
3. Try playing a video (confirms mpv and yt-dlp are accessible)
4. If OAuth is configured, sign in and verify subscriptions load
5. Check the sidebar footer shows "ElecTube v1.0"
